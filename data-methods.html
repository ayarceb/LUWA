LUWA integrates satellite observations, atmospheric transport modelling and lightweight emission-estimation methods. The workflow combines TROPOMI-based products, meteorological data, data assimilation and LOTOS-EUROS simulations to support source-level environmental analysis.

Data Assimilation

Integration of observations and model outputs to improve the representation of atmospheric composition, transport and source influence.

Satellite Observations

Use of satellite products, including TROPOMI-based observations, to detect spatial patterns of NO2, SO2, NH3 and related atmospheric signals.

Emission Estimation

Application of lightweight methods and inverse-modelling concepts to estimate emissions from point sources, industrial areas and agricultural systems.

LOTOS-EUROS on AWS

Atmospheric chemistry-transport simulations using LOTOS-EUROS workflows deployed in cloud infrastructure for controlled environmental scenarios.

Plume Diagnostics

Analysis of plume direction, dispersion, source attribution and meteorological drivers using satellite data and wind fields.

Computational Planning

Configuration of simulation domain size, grid resolution, simulation period and output cadence for cloud-based atmospheric modelling experiments.

TNO LOTOS-EUROS IMI System

LOTOS-EUROS simulation: input → output table

This section explains the simulation workflow from the client request to the delivered products. Humath provides the domain, grid resolution, species of interest, output type and output cadence, then sends the simulation request. LOTOS-EUROS uses meteorology, emissions, boundary conditions, land cover and surface data as model inputs, computes atmospheric transport, chemistry and deposition, and returns structured outputs through LUWA. For both available domains, the output grid is configurable up to 1 km grid spacing. The input datasets have their own native spatial resolution and temporal variability; they are documented and mapped to the selected model grid for each run.

Resolution rule for Humath: the LOTOS-EUROS output resolution is configurable up to 1 km in both available domains. This does not mean that every input layer is native 1 km. Meteorology, boundary conditions, emissions, land cover, fires and surface fields enter with different native resolutions and update frequencies. The run package should therefore report both: the configured output grid and the native resolution of each input field.

What it means for Humath

Domain selection

Humath selects the simulation domain: Europe / MACC domain or Mexico pilot box.

Resolution and period

Humath selects the grid resolution, simulation dates and output cadence: hourly or 3-hourly.

Species of interest

Humath selects the pollutants and diagnostics required for the client: gases, aerosols, deposition or plume products.

Output type and launch

Humath selects the output format and sends the simulation request. LUWA returns files, maps, summaries or API-ready layers.

1. What Humath configures

Domain, resolution, simulation period, species of interest, output type and output cadence.

2. LOTOS-EUROS input

Meteorology, emissions, boundaries, land cover, topography, soil, sea mask, fires and chemistry settings.

3. LOTOS-EUROS output

Concentration fields, deposition fields, maps, NetCDF/CSV/GeoJSON products, QA/QC summaries and API-ready datasets.

Europe / MACC domain

Simulation window: 2024-08-12 to 2024-08-22.

  • Preset: MACC / default LOTOS-EUROS European domain.
  • Bounds: West 15.0°W, East 35.0°E, South 35.0°N, North 70.0°N.
  • Preset grid: 0.50° longitude × 0.25° latitude; 100 × 140 cells.
  • Configurable output resolution: up to 1 km grid spacing.
  • Output cadence: hourly or 3-hourly.

Mexico pilot box

Simulation window: 2024-07-01 to 2024-07-27.

  • Provided pilot box: North 24.0°N, South 23.0°N, West 103.0°W, East 102.0°W.
  • Configurable output resolution: selected by Humath, up to 1 km grid spacing.
  • Setup: non-European domain using global meteorology, global boundaries and global land-cover support.
  • Output cadence: hourly or 3-hourly.

Output principle: LOTOS-EUROS performs the atmospheric simulation. LUWA packages the complete input-output chain: documented LOTOS-EUROS inputs, model-computed atmospheric fields, interpretable diagnostics and delivery-ready products for Humath.

Items to configure before production launch: output grid resolution up to 1 km, species of interest, output type, output cadence, Mexico anthropogenic inventory, inventory year, boundary-condition product/version, spin-up or restart strategy, active natural-emission modules, fire/dust treatment and the native-resolution manifest for every input product.

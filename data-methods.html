LUWA integrates satellite observations, atmospheric transport modelling and lightweight emission-estimation methods. The workflow combines TROPOMI-based products, meteorological data, data assimilation and LOTOS-EUROS simulations to support source-level environmental analysis.

Data Assimilation

Integration of observations and model outputs to improve the representation of atmospheric composition, transport and source influence.

Satellite Observations

Use of satellite products, including TROPOMI-based observations, to detect spatial patterns of NO2, SO2, NH3 and related atmospheric signals.

Emission Estimation

Application of lightweight methods and inverse-modelling concepts to estimate emissions from point sources, industrial areas and agricultural systems.

LOTOS-EUROS on AWS

Atmospheric chemistry-transport simulations using LOTOS-EUROS workflows deployed in cloud infrastructure for controlled environmental scenarios.

Plume Diagnostics

Analysis of plume direction, dispersion, source attribution and meteorological drivers using satellite data and wind fields.

Computational Planning

Estimation of simulation domain size, CPU needs and operational feasibility for cloud-based atmospheric modelling experiments.

TNO LOTOS-EUROS IMI System

LUWA simulation capability: input → output table

This section explains the simulation as a controlled transformation: data enters LUWA, LOTOS-EUROS performs transport, chemistry and deposition calculations, and Humath receives usable outputs through files, reports, dashboards or API-ready products. For both available domains, the finest spatial resolution available is 1 km grid spacing, subject to input-data availability and computing budget.

What it means for Humath

Clear data flow

Inputs are documented before the run, and outputs are delivered with metadata, dates, domain and resolution.

Decision-ready products

Humath receives maps, time series, model fields, diagnostics and run summaries, not raw internal model complexity.

Protected LUWA backend

Humath consumes results without access to source code, credentials, containers, model internals or proprietary configuration.

Configurable delivery

Outputs can be hourly or 3-hourly, depending on the analysis need, storage volume and API cadence.

1. What feeds LUWA

Meteorology, emissions, boundaries, land cover, topography, soil, sea mask, fires and chemistry settings.

2. What LUWA computes

Transport, chemical transformation, aerosol partitioning, dry deposition, wet removal and source influence.

3. What Humath receives

Model outputs, maps, NetCDF/CSV/GeoJSON products, QA/QC summaries and API-ready datasets.

  • Finest available resolution: 1 km grid spacing, when selected for a high-resolution run.

Mexico pilot box

Simulation window: 2024-07-01 to 2024-07-27.

  • Provided pilot box: North 24.0°N, South 23.0°N, West 103.0°W, East 102.0°W.

  • Setup: non-European domain using global meteorology, global boundaries and global land-cover support.
  • Output cadence: hourly or 3-hourly.

Output principle: LUWA does not only launch a model run. It packages the complete input-output chain: documented inputs, model-computed atmospheric fields, interpretable diagnostics and delivery-ready products for Humath.

Items to define before production launch: selected grid resolution within the available range up to 1 km, Mexico anthropogenic inventory, inventory year, boundary-condition product/version, spin-up or restart strategy, output cadence, active natural-emission modules and whether fire/dust influence should be treated as a central result or as context.
